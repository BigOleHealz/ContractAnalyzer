# Test Issues Guidelines

## Purpose
Test issues are created to verify workflows, automations, and integrations without impacting real project data or metrics.

## Creating a Test Issue
1. Go to the Issues tab in the repository.
2. Click on "New issue".
3. Select the "Test Issue" template.
4. Fill in the necessary details and submit.

## Important Notes
- Test issues should always have the `test` label.
- They are excluded from triggering workflows meant for real issues.
- Ensure test issues do not contain sensitive or real project data.

## Usage
Test issues are ideal for new contributors to practice and for developers to test new features or configurations.
