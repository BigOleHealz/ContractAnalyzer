# Test Issue Creation

This document explains the purpose and usage of the test script for programmatically creating issues.

## Purpose

The test script is designed to simulate real-world scenarios where issues are generated by external systems or automation tools. It helps ensure that programmatic issue creation works reliably and prevents regressions in future updates.

## Usage

1. Ensure that the `GITHUB_TOKEN` environment variable is set with the appropriate permissions to create issues in the repository.
2. Run the script using Node.js:
   ```bash
   node scripts/create_test_issue.js
   ```
3. The script will output the URL of the created issue if successful.
